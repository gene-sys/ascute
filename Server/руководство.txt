Руководство о занесении сведений для обновления данных

1. Создаем запрос для передачи данных из Access в MySQL
1.1. Таблицы начинающиеся со слова "Суточная" передаются в табл. Teplo_sn
1.2. Таблицы начинающиеся со слова "Часовая" передаются в табл. Teplo_hn
1.3. В некоторых таблицах параметр "время простоя" может быть НС, а в других ТНС
1.4. Для контроля передаваемых строк в поле ДатаВремя на условие отбора ставим следующее условие: 
>(SELECT Max(data) FROM teplo_Xn WHERE kod_uzl=NN), где X=h для часовых арх., X=s для суточных арх., NN - код узла.
2. Сведения о созданном запросе записываем в файл исполняемых шаблонов
2.1. Файл исполняемых шаблонов называется pattern.ini и находится в дирректории исполняемого модуля server.exe.
2.2. Предыдущуюю версию перед ее изменением лучше сохранить в другую дирректорию.
2.3. В открытом для изменения файле находим заголовок (он взят в рамки []) соответствующий названию таблицы Access, для которой был создан запрос на обновление, и добавляем параметр Update с именем запроса, напр.: Update=имя запроса.
3. Исполнение обновления текущих данных
3.1. Сведения о запросах обновления текущих данных также записываются в файл pattern.ini.
3.2. Выполните п.2.2.
3.3. В открытом для изменения файле находим заголовок [Текущие].
3.4. В найденом заголовке в параметр "Всего" добавляем значение увеличенное на 1.
3.5. И под новым номером в конце раздела добавляем запрос, напр.:
	[Текущие]
	Всего=3
	...
	3=имя запроса
